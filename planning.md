# 班味浓度检测仪 - 策划案

> 版本：v1.0  
> 日期：2026-07-04  
> 状态：策划稿  
> 形态：AI H5 梗向小游戏 / 可分享结果卡

## 1. 项目概述

### 1.1 一句话概念

用户通过回答一组职场语境选择题，检测自己今天的“班味浓度”，最后获得一张可转发的抽象鉴定报告，例如“班味 87%，已生成工位腌制证明”。

### 1.2 梗核

不是你成熟了，是班味先到了。

“班味”指人在长期上班、开会、同步、对齐、推进、闭环之后，语言、表情和精神状态逐渐被工作场景腌入味的程度。本项目把这种状态做成一个轻量检测仪，用夸张但温和的方式让用户自嘲、共鸣、转发给同事。

### 1.3 产品定位

这是一个 1-3 分钟完成的 H5 小游戏，重点不是严肃测评，而是制造“这不就是我吗”的截图传播。用户不需要登录、不需要输入文字、不需要上传真实聊天记录，只通过固定选项完成检测。

### 1.4 交互原则

首版采用“全程零输入”的选择式玩法：

- 不提供自由文本输入框。
- 不要求用户上传聊天截图或聊天记录。
- 不收集公司、姓名、岗位、同事名称等真实信息。
- 用户只需要点击选项，系统根据选择路径生成最终鉴定。
- 结果页产物命名为“鉴定报告”，强化仪式感和截图传播。

### 1.5 目标用户

| 用户类型 | 描述 | 主要动机 |
|---|---|---|
| 上班族 | 经常开会、写日报、回消息、对接需求的人 | 想用轻松方式表达疲惫和自嘲 |
| 职场内容创作者 | 做小红书、公众号、短视频职场内容的人 | 需要有传播性的互动素材 |
| 同事/朋友小群成员 | 常在群里分享职场梗的人 | 想转发给具体某个人，制造互动 |
| H5 小游戏玩家 | 喜欢快速测评和结果卡的人 | 想获得一个有趣身份标签 |

### 1.6 成功标准

| 指标 | 目标 |
|---|---|
| 完成率 | 进入答题页的用户中，70% 以上完成检测 |
| 分享意愿 | 结果页用户中，30% 以上点击保存或分享 |
| 单局时长 | 60-180 秒 |
| 结果共鸣 | 结果卡文案具备“想发给同事”的明确对象感 |
| 复玩率 | 用户愿意切换答案再测一次，查看不同结果 |

## 2. 核心体验

### 2.1 用户旅程

1. 进入首页，看到“班味浓度检测仪”与一句开场文案。
2. 点击开始检测，进入 8 道职场情境选择题，全程只做选择。
3. 每道题给出一个高代入场景，用户选择自己最可能的反应。
4. 系统实时累积“班味值”和若干隐藏梗标签。
5. 答题结束后播放短暂扫描动画，显示检测报告。
6. 用户获得“班味鉴定报告”，可保存、复制文案或重新检测。

### 2.2 首页开场

主标题：班味浓度检测仪  
副标题：上传不了灵魂也没关系，回答几道题就知道你今天被工位腌到几成熟。  
按钮：开始检测

可选小字：

- 本检测无需输入文字，不读取真实聊天记录。
- 数据仅供自嘲，不构成职业评价。

### 2.3 核心循环

题目场景 -> 选择反应 -> 增加班味值 -> 生成隐藏梗标签 -> 得出浓度等级 -> 产出班味鉴定报告 -> 分享/重测。

### 2.4 体验关键词

荒诞、精准、轻微嘴损、不过度攻击、结果卡强截图感。

### 2.5 网感命名原则

标签不能像后台字段，必须像用户愿意截图认领的“精神工种”。命名遵循三个标准：

- 能当评论发出去：例如“收到学十级选手”“人在下班，魂在飞书”。
- 有具体画面：不要只写“同步依赖”，要写“周末约饭也想拉群对齐”。
- 轻损但不冒犯：笑的是状态，不评价职业能力和人格。

## 3. 玩法设计

### 3.1 题目结构

每局 8 题，题目从固定题库中抽取。每题 4 个选项，对应不同班味分值和隐藏梗标签。用户不输入任何文字，只在预设选项中选择最像自己的反应。

题目类型建议：

| 类型 | 说明 | 示例 |
|---|---|---|
| 职场话术题 | 选择最像自己的回复 | “方便同步一下吗？” |
| 情绪压制题 | 面对突发需求如何反应 | “我这边先看一下。” |
| 会议残留题 | 对开会场景的条件反射 | “这个点我们先拉齐。” |
| 边界感题 | 是否会体面拒绝无效沟通 | “这个需求需要重新确认优先级。” |
| 下班幻觉题 | 是否还能自然进入生活状态 | “我先处理一下，马上就好。” |

### 3.2 班味维度

总分为 0-100，称为“班味浓度”。每个选项同时影响 5 个维度。

| 计算键名（不对用户展示） | 对外梗标签 | 说明 | 高分表现 |
|---|---|---|---|
| 同步依赖 | 万物皆可拉齐 | 习惯把所有事情都拖进群聊和会议 | 周末约饭也想先对齐时间窗口 |
| 闭环执念 | 不闭环会睡不着 | 对确认、推进、反馈有条件反射 | 对方说“之后再说”会浑身难受 |
| 体面疲惫 | 收到学十级 | 把崩溃翻译成稳定、客气、专业 | 嘴上“收到”，灵魂正在强制关机 |
| 需求耐受 | 接锅自动挡 | 面对临时变化时优先接住再说 | 锅刚出现，手已经伸出去了 |
| 工位腌制 | 人在下班魂在飞书 | 生活语言也开始被工作腌入味 | 点奶茶都想确认糖冰策略 |

### 3.3 选项分值

每题 4 个选项建议使用以下强度：

| 选项类型 | 分值 | 文案气质 |
|---|---:|---|
| 野生自然人 | 0-5 | 还会直接说人话 |
| 工位轻腌 | 6-10 | 有一点客气反射 |
| 钉味初显 | 11-15 | 明显开始职场话术化 |
| 班味包浆 | 16-20 | 像被办公软件接管了语言系统 |

### 3.5 结果等级

| 浓度 | 等级 | 结果标题 | 说明 |
|---:|---|---|---|
| 0-20 | 野生自然人 | 工牌还没腌进灵魂 | 你仍然能用人话表达需求，这很珍贵，先别声张。 |
| 21-40 | 轻腌打工人 | 班味刚挂边 | 你偶尔会说“我看一下”，但目前还能靠下班自救。 |
| 41-60 | 工位半熟人 | 已进入同步半径 | 你开始用开会语言处理生活问题，约饭都像开项目启动会。 |
| 61-80 | 班味包浆体 | 嘴上收到，心里离职 | 你稳定、体面、疲惫，已经能自动生成三版委婉回复。 |
| 81-100 | 工位腌入味 | 人在下班，魂在飞书 | 你不是在上班，就是在准备上班后的善后，建议今晚强制断网。 |

### 3.6 隐藏身份

根据最高维度生成一个“班味身份”，让结果更有分享点。

| 触发条件 | 鉴定身份 |
|---|---|
| 万物皆可拉齐最高 | 同步教教主 |
| 不闭环会睡不着最高 | 闭环成瘾患者 |
| 收到学十级最高 | 已读乱回但很体面选手 |
| 接锅自动挡最高 | 锅来我先热一下型人格 |
| 人在下班魂在飞书最高 | 下班未遂常驻嘉宾 |

### 3.7 鉴定报告结构

结果卡必须像一张“鉴定报告”，方便截图传播。

包含字段：

- 标题：班味鉴定报告
- 浓度百分比：例如 87%
- 评级：班味溢出
- 身份：锅来我先热一下型人格
- 症状描述：三句话以内
- 今日处方：一个轻量建议
- 证书编号：BW-日期-随机码
- 分享语：我被鉴定为 87% 工位腌入味，你也来看看自己熟到几分

鉴定报告文案样例：

```text
班味浓度：87%
评级：班味溢出
身份：锅来我先热一下型人格

症状：
你已经能把“我不想干了”翻译成“这个优先级需要重新评估”。
遇到突发需求时，你的第一反应不是拒绝，而是打开日历。
生活里的小事也开始被你拆成目标、路径和验收标准。

今日处方：
允许自己用一句人话结束今天：不行，我要下班了。
```

## 4. AI 角色设计

### 4.1 AI 角色

AI 扮演“班味质检员”，语气像一个冷静、损但不恶毒的抽象检测员。

角色设定：

- 不做心理咨询师，不做职业导师。
- 不判断用户是否优秀，只判断“班味浓不浓”。
- 文案要精准、轻松、有梗，但避免攻击具体职业、公司、性别、地域。

### 4.2 AI 使用位置

MVP 阶段可以不依赖实时 AI，先用规则和文案库生成结果。后续版本即使接入 AI，也只基于用户的选择路径生成鉴定文案，不开放自由输入。

| 场景 | MVP | 后续增强 |
|---|---|---|
| 题目生成 | 固定题库 | 按行业切换预设题库 |
| 结果描述 | 模板拼接 | AI 基于选择路径润色鉴定 |
| 分享语 | 固定文案池 | AI 基于鉴定结果生成转发文案 |
| 每日更新 | 手动维护 | AI 扩写每日题库 |

## 5. 页面与交互

### 5.1 页面列表

| 页面 | 作用 | 核心元素 |
|---|---|---|
| 首页 | 建立梗和动机 | 标题、开场文案、开始按钮、隐私提示 |
| 答题页 | 核心玩法 | 题目、选项、进度条、轻量反馈 |
| 扫描页 | 制造仪式感 | 雷达扫描、百分比跳动、检测文案 |
| 结果页 | 传播核心 | 鉴定报告卡、保存按钮、重测按钮、复制文案 |

### 5.2 交互细节

- 答题时不展示具体加分，避免用户功利选择。
- 每题选择后给一句短反馈，例如“检测到：周末也想拉群对齐”“警报：收到学自动挡已启动”。
- 答完后播放 1.5-2 秒扫描动画，增强期待感。
- 结果页默认展示“班味鉴定报告”卡片，按钮放在卡片下方。
- 支持重新检测，不要求刷新页面。

### 5.3 视觉风格

整体美术风格确定为“伪检测仪 + 办公室荒诞感 + 社交截图感”。画面要像一台不太正经但非常认真工作的班味质检设备：有仪器读数、打印报告、工牌证件、复印机噪点和办公室小物，但整体保持轻松、干净、可分享，不做阴沉压抑的社畜苦情风。

核心视觉判断：

- 第一眼要知道这是一个“检测仪”，不是普通问卷。
- 第二眼要觉得它在认真胡说，像真的生成了一份班味鉴定。
- 第三眼要适合截图转发，结果卡比页面装饰更重要。

| 元素 | 建议 |
|---|---|
| 主色 | 荧光绿 `#39FF88` 作为检测高光，工牌蓝 `#2F6BFF` 作为信息色，警示橙 `#FF9F1C` 作为结果和按钮点缀 |
| 底色 | 近黑蓝灰 `#111827` 或冷灰白 `#F4F7F9` 二选一；MVP 建议深色仪器界面，结果卡使用浅色纸质报告 |
| 背景 | 办公桌俯视、扫描网格、复印机噪点、检测面板、打印纸纹理 |
| 字体气质 | 标题可用较硬朗的黑体，正文使用清晰系统 UI 字体；数字、编号、百分比使用等宽字体强化仪器感 |
| 动效 | 扫描线、进度条、百分比跳动、轻微故障闪烁、报告打印、印章盖下 |
| 图形 | 工牌、咖啡、会议气泡、日历、打卡机、键盘、红点消息、打印纸、检测雷达 |
| 质感 | 半透明仪器面板 + 纸质报告卡；少量噪点和像素扫描纹，不做复杂插画堆叠 |
| 圆角 | 面板 8px 以内，结果卡 6px 左右，按钮偏直角，避免过度可爱化 |

视觉参考方向：

- 仪器 UI：雷达屏、检测读数、状态灯、编号、警示标签。
- 办公室物件：工牌、咖啡杯、键盘、日历、打印机、会议便签。
- 社交传播：结果卡像一张“荒诞证书”或“体检报告”，信息清楚，手机截图完整。

不建议的方向：

- 不做赛博朋克霓虹大场景，避免主题被“酷炫科技”抢走。
- 不做写实真人员工图，避免具体职业、性别、年龄或公司联想。
- 不做沉重丧气插画，整体是自嘲和共鸣，不是压迫感控诉。
- 不使用真实公司 logo、办公软件 logo 或过度相似的品牌界面。

### 5.4 配图与素材清单

MVP 可以先用轻量插画、图标和纹理完成，不必做大量角色图。所有配图都服务“检测仪”和“可截图结果卡”，优先保证首屏、扫描页、结果卡三处有明确视觉记忆点。

| 位置 | 配图内容 | 形式 | 作用 | 优先级 |
|---|---|---|---|---|
| 首页首屏 | 桌面上的“班味检测仪”，屏幕显示扫描网格、工牌、咖啡和红点消息 | 主视觉插画或生成图 | 建立世界观，让用户一眼知道不是普通问卷 | Must |
| 答题页顶部 | 小型检测面板，包含当前题号、状态灯、微型波形 | UI 图形组件 | 强化过程感，不遮挡题目和选项 | Must |
| 选项反馈 | 工牌、咖啡、日历、会议气泡、红点消息等小图标 | 线性图标或双色小贴纸 | 让每题反馈更有记忆点 | Should |
| 扫描页 | 雷达扫描圆盘、进度条、故障噪点、检测光束 | 动效背景或 CSS/Canvas 组件 | 制造仪式感和等待预期 | Must |
| 结果页 | 纸质“班味鉴定报告”卡，带印章、编号、百分比仪表 | 可保存卡片模板 | 传播核心，优先级最高 | Must |
| 高班味结果 | 警示橙印章、过载仪表、红点爆表状态 | 结果卡变体 | 让高分结果更好笑、更想分享 | Should |
| 低班味结果 | 绿色合格章、干净工牌、低电量提示 | 结果卡变体 | 避免低分结果无聊 | Should |
| 分享封面 | 方形封面图，标题为“班味浓度检测仪”，展示检测仪主视觉 | 分享卡图 | 用于社交平台预览和落地页封面 | Could |

页面配图要求：

- 首页主视觉占首屏 35%-45% 高度，移动端不压住标题和开始按钮。
- 答题页配图只做轻量 HUD，不要每题都放大插画，避免拖慢节奏。
- 扫描页可以使用动态网格和雷达光，不需要复杂图片。
- 结果页必须优先为手机截图设计，卡片比例建议 `3:4` 或 `9:16` 内居中展示。
- 保存图片时隐藏页面按钮，只保留报告卡、二维码或入口提示。

结果卡美术模板建议：

| 模板 | 视觉特征 | 适用结果 |
|---|---|---|
| 检测报告版 | 白底纸张、灰色表格、编号、检测章、签发日期 | 默认模板，适合所有分数 |
| 工牌证件版 | 蓝白工牌、头像占位替换为抽象仪表、挂绳元素 | 中高班味身份类结果 |
| 过载警报版 | 深色仪器屏、橙色警告条、百分比爆表、故障噪点 | 81-100 高班味结果 |
| 下班复健版 | 浅色处方单、绿色合格章、轻量生活化图标 | 0-40 低班味结果 |

素材规格：

- 首页主视觉：`1200x900`，透明或深色背景均可，需能裁切到移动端竖屏。
- 分享封面：`1080x1080`，标题清晰，主体在中心安全区。
- 结果卡导出：建议 `1080x1440` 或 `1080x1920`，文字在 `64px` 安全边距内。
- 图标：统一线宽，建议 `24px`、`32px`、`48px` 三档，支持深浅两套颜色。
- 纹理：复印机噪点、扫描线、纸张纹理应可用 CSS 或小尺寸 PNG 平铺实现。

## 9. 验收标准

### F-001 首页展示

AC-F001-01  
Given 用户打开游戏链接  
When 页面加载完成  
Then 用户能看到“班味浓度检测仪”标题、开场文案和“开始检测”按钮。

AC-F001-02  
Given 用户在移动端访问首页  
When 屏幕宽度较窄  
Then 标题、按钮和提示文案不重叠、不被截断。

AC-F001-03  
Given 用户点击“开始检测”  
When 点击成功  
Then 页面进入第一道题。

### F-002 答题流程

AC-F002-01  
Given 用户进入答题页  
When 当前题目展示  
Then 页面显示题干、4 个选项和当前进度。

AC-F002-02  
Given 用户选择一个选项  
When 系统记录答案  
Then 自动进入下一题或显示下一题按钮。

AC-F002-03  
Given 用户完成最后一题  
When 答案记录完成  
Then 系统进入检测计算或扫描过场。

### F-003 计分系统

AC-F003-01  
Given 用户完成 8 道题  
When 系统计算分数  
Then 生成 0-100 之间的班味浓度。

AC-F003-02  
Given 用户选择了不同分值的选项  
When 计算结果  
Then 高班味选项越多，总浓度越高。

AC-F003-03  
Given 某个隐藏维度得分最高  
When 生成结果  
Then 系统匹配对应隐藏身份。

### F-004 扫描过场

AC-F004-01  
Given 用户完成答题  
When 进入扫描页  
Then 页面显示检测动画和状态文案。

AC-F004-02  
Given 扫描动画播放完成  
When 结果已计算  
Then 页面自动进入结果页。

### F-005 结果卡生成

AC-F005-01  
Given 系统已有分数、等级和隐藏身份  
When 进入结果页  
Then 页面展示完整结果卡。

AC-F005-02  
Given 用户班味浓度为 81-100  
When 生成结果卡  
Then 结果等级显示为“班味溢出”。

AC-F005-03  
Given 用户查看结果卡  
When 页面渲染完成  
Then 卡片包含浓度百分比、评级、身份、症状描述和今日处方。

### F-006 重测

AC-F006-01  
Given 用户在结果页  
When 点击“重新检测”  
Then 系统清空本轮答案并返回第一题。

AC-F006-02  
Given 用户重新完成答题  
When 答案不同  
Then 结果可根据新答案变化。

## 10. 边界与风险

### 10.1 内容边界

- 不提供自由输入框。
- 不读取真实聊天记录。
- 不要求用户输入公司、同事、客户姓名。
- 不做真实职业能力评价。
- 不攻击具体公司、职业、性别、地域、年龄群体。
- 不把疲惫、焦虑等状态包装成医学诊断。

### 10.2 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 文案过度刻薄 | 用户感到被攻击 | 保持自嘲和轻损，不做人身评价 |
| 题目太像问卷 | 游戏感不足 | 加入扫描、证书、身份等仪式感 |
| 结果不够准 | 分享欲降低 | 使用隐藏维度生成更具体身份 |
| 截图不好看 | 传播受限 | 结果卡优先设计，保证移动端视觉完整 |
| AI 生成失控 | 输出不当内容 | MVP 先使用固定文案库，AI 仅做后续增强 |

## 11. 开放问题

- 是否需要首版就支持生成图片，还是先用结果页截图替代？
- 题目数量固定 8 道，还是每局从题库随机抽取 8 道？
- 结果卡视觉已确定：MVP 默认使用“检测报告版”，v1.1 再扩展“工牌证件版”“过载警报版”“下班复健版”。
- 是否为不同行业准备差异化题库，例如互联网、广告、设计、销售、乙方？
- 最终产物名称是否统一为“班味鉴定报告”，还是随机使用“工位腌制证明”“下班未遂证明”等标题？

## 12. MVP 开发建议

首版建议控制为纯前端 H5：

1. 本地配置题库、分数、梗标签和鉴定文案。
2. 用户完成 8 道选择题。
3. 系统按规则计算班味浓度和隐藏身份。
4. 生成一张可截图的班味鉴定报告。
5. 支持重测。

暂不做登录、真实聊天上传、自由文本输入、行业题库、AI 实时生成和用户数据留存。第一版的关键不是技术复杂度，而是鉴定报告够准、够好笑、够想发给同事。
